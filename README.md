# Python Selenium Test Automation Framework (Page Object Model)

A robust, enterprise-grade UI automation testing framework built from scratch using **Python**, **Selenium WebDriver**, and **Pytest**. This framework implements the **Page Object Model (POM)** architectural design pattern and features automated HTML reporting, synchronized explicit waits, failure screenshot capturing, and seamless **GitHub Actions CI/CD** pipeline integration.

---

## 🚀 Key Framework Features

* **Page Object Model (POM):** Clean separation of test logic (assertions) and page-specific layout (locators and actions).
* **Robust Synchronization:** Built-in Selenium `WebDriverWait` (Explicit Waits) to eradicate flaky tests caused by network latency or asynchronous loading elements.
* **Custom Test Hook Reporting:** Integrated Pytest lifecycle hooks to automatically capture browser screenshots the exact millisecond a runtime test failure occurs.
* **Continuous Integration (CI/CD):** Integrated GitHub Actions workflow that executes the regression suite automatically in a headless Linux environment on every `push` or `pull_request`.
* **Sleek HTML Dashboards:** Automatic generation of standalone, self-contained HTML execution reports including metrics and status breakdowns.

🛠️ Local System Prerequisites & Execution
1. Installation
Clone the repository to your workstation, configure a virtual workspace, and execute the installation sequence:

# Clone the automation project
git clone [https://github.com/your-username/selenium-pom-framework.git](https://github.com/your-username/selenium-pom-framework.git)
cd selenium-pom-framework

# Install project dependencies
pip install -r requirements.txt

2. Running Tests Locally
Execute the default suite tracking via Pytest commands:
# Run the complete test suite sequentially
pytest -v

# Run tests and generate an interactive standalone HTML summary dashboard
pytest -v --html=reports/dashboard.html --self-contained-html

## 🤖 GitHub Actions CI/CD Engine

The testing pipeline executes on an isolated `ubuntu-latest` runner on every code integration milestone. It orchestrates the following workflow steps automatically:

1. **Checkout Code:** Clones the codebase from GitHub into the virtual machine runner environment.
2. **Python Sandbox setup:** Initializes a stable **Python 3.11** environment and configures pipeline caching.
3. **Browser Provisioning:** Downloads, provisions, and configures the standard production-grade **Google Chrome** software binary package.
4. **Dependency Installation:** Automates package setup via the project dependency list.
5. **Headless Execution:** Forces headless execution conditions (`--headless=new`, `--no-sandbox`) to allow fast runtime performance in windowless cloud environments.
6. **Artifact Archiving:** Publishes an archive of compiled **HTML Reports** directly to your GitHub workflow summary dashboard as downloadable build artifacts.
